# Config1 Gamma参数优化探索脚本

## 功能概述

该脚本专门用于分析Config1配置下的gamma参数设定，实现以下功能：

### 1. 参数优化功能
- 系统性探测gamma参数的最优值
- 采用智能搜索策略，避免盲目遍历
- 支持自定义搜索范围、步长和收敛条件

### 2. 深度控制
- 严格控制深度参数，确保稳定维持在目标深度（默认5）
- 允许±1的深度误差范围
- 自动调整gamma值以逼近目标深度

### 3. 并行计算架构
- 基于multiprocessing.Pool实现并行探索
- 充分利用系统多核处理能力
- 支持自定义并行核心数

### 4. 智能搜索策略
- 采用中心点+步长的迭代搜索
- 动态调整步长（初始0.2，每轮衰减至0.7）
- 基于结果反馈自动调整搜索方向
- 避免盲目遍历，提高搜索效率

### 5. 详细结果记录
- 记录每次探测的参数组合
- 包含性能指标：depth, fitness, f1, rule_count, tcam_pct等
- 记录计算时间，评估搜索效率
- 保存完整的match_config信息

### 6. 收敛条件
- 深度误差≤tolerance（默认0.1）
- 步长<0.05时自动停止
- 达到最大迭代次数（默认10）
- 找到可行解且满足深度要求

### 7. 可视化报告
- 输出优化过程的详细报告（JSON格式）
- 显示gamma参数影响趋势表
- 统计信息：总测试数、可行解数、总耗时、平均耗时
- 推荐最优gamma值及区间

### 8. 可配置性
- 支持命令行参数配置所有关键参数
- 可调整：配置名称、目标深度、搜索范围、步长、并行度、收敛条件
- 支持自定义输出文件路径

## 使用方法

### 基本用法
```bash
# 使用默认参数优化Config1的gamma
python gamma_optimizer.py

# 指定输出文件
python gamma_optimizer.py --output my_report.json
```

### 高级用法
```bash
# 自定义搜索范围
python gamma_optimizer.py --gamma-min 0.8 --gamma-max 2.0

# 自定义目标深度
python gamma_optimizer.py --target-depth 6

# 调整并行核心数
python gamma_optimizer.py --num-cores 8

# 自定义收敛条件
python gamma_optimizer.py --max-iterations 15 --tolerance 0.05
```

### 完整参数列表
```
--config              配置名称（默认：Config1）
--target-depth        目标深度（默认：5）
--gamma-min           gamma最小值（默认：0.5）
--gamma-max           gamma最大值（默认：3.0）
--initial-step      初始步长（默认：0.2）
--num-cores          并行核心数（默认：4）
--max-iterations     最大迭代次数（默认：10）
--tolerance         收敛容差（默认：0.1）
--output             输出文件路径（默认：自动生成）
```

## 输出说明

### 控制台输出
- 实时显示探索进度
- 每次迭代显示当前中心点和步长
- 显示最优解发现和收敛状态
- 最终显示优化报告摘要和趋势分析

### JSON报告文件
包含以下字段：
- `config_name`: 配置名称
- `target_depth`: 目标深度
- `optimization_params`: 优化参数配置
- `final_result`: 最终最优结果
- `statistics`: 统计信息
- `exploration_history`: 完整探索历史

## 技术特点

1. **自适应搜索**: 根据每次迭代结果动态调整搜索中心和步长
2. **并行加速**: 充分利用多核CPU，显著缩短搜索时间
3. **智能收敛**: 基于深度误差和步长双重收敛判断
4. **完整记录**: 保存所有测试结果，便于后续分析
5. **可视化报告**: 提供趋势分析和推荐值
6. **高度可配置**: 支持所有关键参数的命令行配置

## 注意事项

1. 确保已安装numpy库：`pip install numpy`
2. 建议在空闲时运行，避免影响其他任务
3. 可根据实际需求调整并行核心数，建议不超过物理核心数
4. 搜索范围和步长需要根据实际gamma敏感度调整
5. 收敛容差越小，精度越高但可能需要更多迭代
