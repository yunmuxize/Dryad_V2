# 三模型特征使用总结

## 1. IIsy (原作)

**数据集场景**: IoT 设备流量分类

**特征列表** (共 11 个特征):
1. `frame_len` - 帧长度
2. `eth_type` - 以太网类型
3. `ip_proto` - IP 协议类型
4. `ip_flags` - IP 标志位
5. `ipv6_nxt` - IPv6 下一个头部
6. `ipv6_opt` - IPv6 选项
7. `tcp_srcport` - TCP 源端口
8. `tcp_dstport` - TCP 目的端口
9. `tcp_flags` - TCP 标志位
10. `udp_srcport` - UDP 源端口
11. `udp_dstport` - UDP 目的端口

**特点**: 
- 侧重于网络层和传输层特征
- 同时支持 IPv4 和 IPv6
- 包含端口和协议信息用于 IoT 设备识别

---

## 2. NetBeacon (ML2Switch 实现)

**数据集场景**: UNSW-NB15 流量分类

**特征列表** (共 10 个特征):
1. `f0`: `protocol` (hdr.ipv4.protocol) - IP 协议类型 [8 bits]
2. `f1`: `ihl` (hdr.ipv4.ihl) - IP 头部长度 [4 bits]
3. `f2`: `tos` (hdr.ipv4.tos) - 服务类型 [8 bits]
4. `f3`: `flags` (hdr.ipv4.flags) - IP 标志位 [3 bits]
5. `f4`: `ttl` (hdr.ipv4.ttl) - 生存时间 [8 bits]
6. `f5`: `dataOffset` (meta.dataOffset) - TCP 数据偏移 [4 bits]
7. `f6`: `flags` (meta.flags) - TCP 标志位 [8 bits]
8. `f7`: `window` (meta.window) - TCP 窗口大小 [16 bits]
9. `f8`: `udp_length` (meta.udp_length) - UDP 长度 [16 bits]
10. `f9`: `totalLen` (hdr.ipv4.totalLen) - IP 总长度 [16 bits]

**特点**:
- 专注于 IPv4 特征
- 包含 IP 层、TCP 层、UDP 层的关键字段
- 特征位宽明确定义，便于硬件实现

---

## 3. Planter (ML2Switch 实现)

**数据集场景**: UNSW-NB15 流量分类

**特征列表** (共 10 个特征):
1. `f0`: `srcPort` (meta.srcPort) - 源端口 [16 bits]
2. `f1`: `dstPort` (meta.dstPort) - 目的端口 [16 bits]
3. `f2`: `protocol` (hdr.ipv4.protocol) - IP 协议类型 [8 bits]
4. `f3`: `ihl` (hdr.ipv4.ihl) - IP 头部长度 [4 bits]
5. `f4`: `tos` (hdr.ipv4.tos) - 服务类型 [8 bits]
6. `f5`: `ttl` (hdr.ipv4.ttl) - 生存时间 [8 bits]
7. `f6`: `dataOffset` (meta.dataOffset) - TCP 数据偏移 [4 bits]
8. `f7`: `window` (meta.window) - TCP 窗口大小 [16 bits]
9. `f8`: `udp_length` (meta.udp_length) - UDP 长度 [16 bits]
10. `f9`: `totalLen` (hdr.ipv4.totalLen) - IP 总长度 [16 bits]

**特点**:
- 与 NetBeacon 高度相似
- 主要区别：包含了**源端口和目的端口**特征（f0, f1）
- 缺少 NetBeacon 中的 `ip_flags` (f3) 和 `tcp_flags` (f6)

---

## 4. ML2Switch 数据加载器特征

**UNSW-NB15 数据集** (`load_unsw_data()` / `load_unsw_pkl_data()`):
```python
["srcPort", "dstPort", "protocol", "ip_ihl", "ip_tos", "ip_ttl", 
 "tcp_dataofs", "tcp_window", "udp_len", "length"]
```

**UNIV 数据集** (`load_univ_data()`):
```python
["srcPort", "dstPort", "protocol", "ip_ihl", "ip_tos", "ip_ttl", 
 "tcp_dataofs", "tcp_window", "udp_len", "length"]
```

## 5. 特征对比总结

| 特征类别 | IIsy | NetBeacon | Planter |
|---------|------|-----------|---------|
| **端口信息** | TCP/UDP 源/目的端口 (4个) | ❌ | 源/目的端口 (2个) |
| **IP 协议** | ✅ | ✅ | ✅ |
| **IP 头部** | IPv4/IPv6 标志、选项 | IHL, ToS, Flags, TTL, TotalLen | IHL, ToS, TTL, TotalLen |
| **TCP 特征** | 源/目的端口, Flags | DataOffset, Flags, Window | DataOffset, Window |
| **UDP 特征** | 源/目的端口 | Length | Length |
| **帧/包长度** | Frame Length | Total Length | Total Length |
| **以太网层** | Eth Type | ❌ | ❌ |
| **IPv6 支持** | ✅ | ❌ | ❌ |

**关键差异**:
1. **IIsy**: 最全面，支持 IPv6，包含以太网层特征
2. **NetBeacon**: 专注于 IPv4，强调 IP 和 TCP 标志位
3. **Planter**: 在 NetBeacon 基础上增加端口信息，移除部分标志位

**ML2Switch 统一化**:
- 在 ML2Switch 框架中，NetBeacon 和 Planter 都使用 UNSW-NB15 数据集
- 特征集基本一致，主要差异在于端口特征的使用
- 所有特征都被映射到 P4 可编程交换机的 PHV (Packet Header Vector) 字段
