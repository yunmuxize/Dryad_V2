- **问题**: 参数组合数计算错误（使用了sum而不是product）

## 解决方案

### 1. 修复参数组合数计算

已修复：从 `sum(len(v))` 改为 `reduce(operator.mul, [len(v)])`，正确计算笛卡尔积。

### 2. 启用GPU加速

LightGBM支持GPU训练，可以显著加速：

```bash
# 使用GPU训练
python dryad_predictor_enhanced.py --extensive --gpu

# 或使用优化训练脚本
python scripts/train_optimized.py --gpu
```

**预期加速**: 2-5倍（取决于GPU性能）

### 3. 优化参数网格

如果训练时间仍然过长，可以考虑：
- 减少参数值范围
- 使用更粗粒度的搜索
- 分阶段训练（先粗搜索，再精细搜索）

## 结论

**训练速度慢的主要原因**：

1. **参数组合数大幅增加**：扩展模式是标准模式的**61倍**
2. **Memory模型影响最大**：从972增加到82,944（85.3倍）
3. **新增正则化参数**：`reg_alpha`和`reg_lambda`进一步增加了组合数
4. **参数值范围扩大**：多个参数从2-3个值增加到3-4个值

**建议**：
- 使用GPU加速训练（2-5倍提升）
- 如果时间紧迫，使用标准模式
- 如果需要最佳性能，使用扩展模式+GPU，预计5-10小时
